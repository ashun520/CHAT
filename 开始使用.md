## 🚀 快速启动（3 步走）

### 方法一：使用批处理脚本（推荐，最简单）

#### 1️⃣ 启动数据库（Docker 方式）
```bash
双击运行：start-docker.bat
```
这会自动启动 MongoDB 和 Redis 容器。

#### 2️⃣ 启动后端服务
```bash
双击运行：start-backend.bat
```
看到 "SuperChat 服务器已启动" 表示成功。

#### 3️⃣ 启动前端应用
```bash
双击运行：start-frontend.bat
```
这会自动打开应用。

### 方法二：手动启动

#### 1️⃣ 安装依赖
```bash
# 安装所有依赖
双击运行：install-dependencies.bat
```

#### 2️⃣ 启动 MongoDB 和 Redis
**使用 Docker（推荐）:**
```bash
docker-compose up -d
```

**或本地安装:**
- MongoDB: https://www.mongodb.com/try/download/community
- Redis: https://github.com/microsoftarchive/redis/releases

#### 3️⃣ 启动后端
```bash
cd backend
npm install
npm run dev
```

#### 4️⃣ 启动前端
```bash
cd frontend
flutter pub get
flutter run
```

## 🔧 配置说明

### 后端配置 (.env)
```bash
PORT=3000                          # 服务端口
MONGODB_URI=mongodb://localhost:27017/superchat
REDIS_HOST=localhost
REDIS_PORT=6379
JWT_SECRET=your-secret-key         # JWT 密钥（生产环境请修改）
```

### 前端配置
Android 模拟器：
- API 地址：`http://10.0.2.2:3000/api`
- Socket 地址：`http://10.0.2.2:3000`

真机调试需要修改为电脑 IP：
- 例如：`http://192.168.1.100:3000/api`

## ⚠️ 常见问题

### 1. 后端启动失败
**检查 MongoDB 和 Redis 是否已启动**
```bash
# Docker 方式
docker-compose ps

# 查看日志
docker-compose logs -f
```

### 2. 前端无法连接后端
- 检查后端是否启动（访问 http://localhost:3000/health）
- 检查 API 地址配置是否正确
- 检查防火墙设置

### 3. 端口被占用
修改 `backend/.env` 中的 PORT 值

### 4. Flutter 依赖安装失败
```bash
flutter clean
flutter pub cache repair
flutter pub get
```
